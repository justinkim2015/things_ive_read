# Clean Code

## Bad code
- It slows down productivity in the future.
- You won't go back and fix it, no matter what you promise yourself.

## Attitude
- It is your responsibiity to maintain the code base.  You know the most about the code, therefore you must fight for it's cleanliness.
- All developers face the issue of having deadlines while knowing that previous messes will slow you down in the future.
