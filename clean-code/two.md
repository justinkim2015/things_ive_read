# Meaningful Names

## Use intention revealing names
- The name should shell you why it exists, what it does, and how it's used.

## Avoid Disinformation
- Don't use common abbreviations to mean other things
- Don't name things misleadingly, like naming a Map object a list.

## Make Meaningful Distinctions
- When you have two variables with similar names, make sure you the difference is for a good reason

## Use Pronounceable Names
- Humans think of things in the frame of words.
- If variables aren't pronounceable, they're hard to understand

## Use Searchable Names
- Single letter names etc are hard to find searching across a project.
- The length of a name should correspond to the size of its scope

## Class Names
- Should have nouns or noun phrases
- Avoid verbs

## Method Names
- Should have verb phrases

## Pick One Word per Concept
- If you're getting something, pick a word for it, fetch, get, retrieve etc and be consistent.
